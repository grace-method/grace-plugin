# Cross-Cutting Principles

- Working software is the true test. Analysis and docs support it, not replace it.
- Sequential stages, not walls. Feedback from later stages refines earlier ones.
- Good enough, not perfect. Reach adequacy, not false certainty.
- Rigor scales with risk. Project type, change scale, and predictability shape process.
- Traceability is non-negotiable. Work that does not trace to a goal should be questioned.
- Documentation is a first-class deliverable, not an afterthought.
