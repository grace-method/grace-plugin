# Project Documentation

This folder is the project-local memory for GRACE work. The global runtime guidance lives outside this project in `.claude/` and `.codex/`; this `docs/` folder records the facts, decisions, rationale, artifacts, and evidence for this project.

## Starter Files

- `README.md` explains the documentation structure and helps humans quickly re-orient.
- `HANDOFF.md` records concise continuation context for future sessions.
- `BACKLOG.md` stores deferred work, later-horizon ideas, and revisit triggers.

## Stage Folders

- `release-planning/` — cross-stage release plans that select the current product-graph slice.
- `01-project-definition/` — product definition, initiative framing, target users, value, and scope intent.
- `02-analysis/` — goals, problems, solutions, requirements, risks, and release-horizon scope.
- `03-design/` — UX, system, UI, traceability, and design review artifacts.
- `04-implementation/` — features, stories, technical debt, and implementation notes.
- `05-validation/` — validation evidence and outcome review.
- `06-operations/` — readiness, runbooks, incidents, and operational learning.
- `07-retirement/` — end-of-life planning when relevant.

Detailed stage artifacts should be created by the relevant GRACE skill when that stage begins. Empty templates should not be treated as evidence that the work is done.
